# Documento 07 - Control de versiones del frontend PIAR

## 1. Información general

- Proyecto: PIAR Frontend
- Alcance: control de versiones visible en la estructura del proyecto
- Observación: se evidencia el uso de Git como sistema de control de versiones

## 2. Evidencia del control de versiones

- El repositorio del frontend contiene una carpeta .git.
- Esto indica que el proyecto está bajo control de versiones Git.

## 3. Prácticas observadas

- El proyecto presenta una estructura organizada por carpetas y archivos de documentación.
- Los cambios pueden mantenerse rastreables mediante commits locales.
- La documentación técnica se almacena en la carpeta docs/ para seguimiento del proyecto.

## 4. Limitaciones observadas

- No se encontraron evidencias de políticas formales de ramas ni de flujo de integración continuo en la estructura analizada.
- No se encontraron archivos de configuración de GitHub Actions, GitLab CI u otros flujos automáticos.
- No se encontraron convenciones detalladas de commits o de releases en el repositorio actual.

## 5. Recomendación de gestión

Para un proyecto de tipo educativo y académico, el control de versiones actual es suficiente para mantener trazabilidad básica. No obstante, se recomienda formalizar el uso de ramas, mensajes de commit y registros de cambios en futuras etapas del proyecto.

## 6. Resumen

El frontend oficial está contenido en un repositorio Git, lo que permite control básico de versiones. La estructura actual no demuestra un flujo avanzado de ramas o automatización, pero sí evidencia la existencia de trazabilidad del código y de la documentación.
