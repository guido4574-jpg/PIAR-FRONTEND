# Documento 06 - Librerías utilizadas en el frontend PIAR

## 1. Información general

- Proyecto: PIAR Frontend
- Tipo de desarrollo: frontend estático con JavaScript vanilla
- Observación: el proyecto usa bibliotecas externas cargadas desde CDN

## 2. Librerías y recursos externos

### 2.1 Bootstrap Icons

- Biblioteca usada para íconos visuales en botones, navegación y estados.
- Cargada desde CDN en index.html.

### 2.2 Chart.js

- Biblioteca usada para renderizar gráficos en el dashboard.
- Cargada desde CDN en index.html.

### 2.3 Bootstrap Bundle

- Biblioteca JS del framework Bootstrap cargada desde CDN.
- Se incorpora como recurso adicional al proyecto.

## 3. Tecnologías nativas del navegador utilizadas

Además de las librerías externas, el frontend hace uso de tecnologías y APIs del navegador, entre ellas:

- DOM API
- localStorage
- Fetch API
- FormData
- Blob y URL.createObjectURL para descarga de respaldo

## 4. Dependencias de desarrollo detectadas

No se evidencian archivos de configuración de build, empaquetado o dependencias locales propias del frontend en la estructura analizada. La integración depende de recursos cargados en el navegador.

## 5. Resumen

El frontend oficial usa unas pocas librerías externas para mejorar la interfaz visual y los gráficos, pero su núcleo sigue siendo JavaScript vanilla y APIs del navegador. No se observa un stack complejo ni una cadena de herramientas de compilación definida en el proyecto actual.
